## Discord Configuration

### Register a new client

1. Go to the Discord Developer Portal and choose your application: [https://discord.com/developers/applications](https://discord.com/developers/applications)
2. Click on "OAuth2"
3. Add the redirect uri
 - {your-domain}/ui/login/login/externalidp/callback
 - Example redirect url for the domain `https://acme-gzoe4x.zitadel.cloud` would look like this:  `https://acme-gzoe4x.zitadel.cloud/ui/login/login/externalidp/callback`
4. Click on "Reset Secret" and confirm
5. Save the Client ID and Client Secret
6. Save your changes

### Create a new Discord Provider

The Discord provider template has everything you need preconfigured.
Add the client ID and secret created before in the application OAuth2 settings.

You can configure the following settings if you like, a useful default will be filled if you don't change anything:

**Scopes**: The scopes define which scopes will be sent to the provider, `identify` and `email` are prefilled. This information will be taken to create/update the user within ZITADEL.
